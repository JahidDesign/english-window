Frequently Asked Questions

1. var
Scope: Function-scoped.
Hoisting: Gets hoisted to the top of its scope and initialized with undefined.
Redeclaration: Can be redeclared and updated within the same scope.
Usage: Older way to declare variables. Not recommended in modern JavaScript because it can cause bugs due to its function scope and hoisting behavior.


2. let
Scope: Block-scoped (anything inside {}).
Hoisting: Hoisted but not initialized. Accessing before declaration causes a ReferenceError (Temporal Dead Zone).
Redeclaration: Cannot be redeclared in the same scope, but can be updated.
Usage: Recommended when the value of the variable changes.


3. const
Scope: Block-scoped.
Hoisting: Same as let (hoisted but not initialized).
Redeclaration: Cannot be redeclared or reassigned, but if it's an object/array, properties/elements can be changed.
Usage: Use when you don't want to reassign the variable.


1. map()
Purpose: Creates a new array by applying a function to each element of the original array.
Returns: A new array with the transformed elements.
Use when: You need to transform or modify each element in an array.


2. forEach()
Purpose: Executes a function once for each element in an array, but doesn't return anything.
Returns: Undefined, always.
Use when: You want to do something (like logging or updating an external variable) with each element but don’t need a new array.


3. filter()
Purpose: Creates a new array containing only elements that pass a test (return true in the callback function).
Returns: A new array with filtered elements.
Use when: You want to keep or exclude certain elements based on a condition.

Arrow Functions (=>)
Arrow functions are a simpler way to write functions in JavaScript, introduced in ES6.
✅ Basic Syntax: 


                        // Regular function
                        function add(a, b) {
                          return a + b;
                        }
                        
                        // Arrow function
                        const add = (a, b) => a + b;


✅ Features:

  1. Shorter syntax.
  2. They don’t have their own this, arguments, super, or new.target.
  3. Useful for callback functions, array methods, etc.

JavaScript Promises provide a way to handle asynchronous operations by representing a value that may be available now, in the future, or never. Here’s an overview of how they work:
1. Promise States
A Promise has three states:
1. Pending: The initial state, meaning the operation is still in progress.
2. Fulfilled: Indicates that the operation completed successfully, and the promise now holds a value.
3. Rejected: Means the operation failed, and the promise holds a reason (typically an error) for the failure.

2. Creating a Promise
You can create a promise using the Promise constructor: 

                        
                            const myPromise = new Promise((resolve, reject) => {
                                // Do some asynchronous work...
                                if (/* operation successful */) {
                                  resolve('Success!');
                                } else {
                                  reject('Error occurred');
                                }
                              });

Closures in JavaScript allow a function to access variables from an enclosing scope, even after that outer function has finished executing. This means the inner function "remembers" the environment in which it was created.


How Closures Work
Creation of the Scope: When you define a function inside another function, the inner function has access to the outer function’s variables and parameters.
Retaining Access: Even after the outer function completes execution, the inner function continues to hold a reference to the outer function's scope. This preserved scope is what we call a "closure".
Practical Use:
Data Encapsulation: Closures enable the creation of private variables and methods.
Callbacks and Asynchronous Code: They allow you to use data from a specific context in asynchronous functions like event handlers or timers.
